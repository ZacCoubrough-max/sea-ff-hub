Future Considerations
Planned features and enhancements for the SEA Fulfilment Hub
✦
AI-Powered Summaries
Generate plain-English performance summaries from your live data. Choose from weekly recap, action items, movers & trends, or stakeholder email format. One click, ready to share.
Requires Netlify Edge Function setup for CORS. API key stored securely as environment variable.
📈
Year-on-Year Comparison
Compare the same week or quarter across years. W07 2026 vs W07 2025 side by side, per store. The data structure already supports it — just needs a YoY view layer.
Available once 12+ months of data has been loaded. Recommended to compare at KPI level rather than composite scores.
🔍
Store Benchmarking
Cross-country benchmarking to compare equivalent stores in MY, SG and TH. Identify which market sets the standard and share best practice across the SEA network.
Requires consistent store ID mapping across markets.
🔔
Automated Alerts & Digest
Weekly email or Teams digest sent automatically after upload. Highlights Tier D stores, missed revenue, and biggest movers without needing to open the dashboard.
Requires a webhook or scheduled function — Netlify or a simple automation tool like Zapier.
🎯
Store Targets & Recovery Plans
Set individual store targets beyond the network-wide claimed target. Track recovery plans for Tier D stores with a target score and deadline, visible in the store drawer.
Would extend the cnc_notes and sfs_notes tables with target and deadline columns.
📋
Programme Expansion
Add additional fulfilment programmes beyond SFS and C&C — for example Same Day Delivery or Returns performance — using the same country tab and scoring architecture.
Each new programme would need its own Supabase tables and CSV parser.
How to request a feature
Raise it in your next session with Claude. The dashboard codebase is preserved across sessions — any of the above can be built incrementally without rebuilding from scratch. The data structure and Supabase setup are already in place for most features.
